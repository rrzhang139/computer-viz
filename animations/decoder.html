where we are: from a number to a wire

Up to now, the tower has been about computing and remembering bits. This page introduces a new job: routing. A CPU stores everything compactly as binary numbers — "register 2," "operation 3" — but hardware can only act on physical wires, one per target. Something has to translate a number into "this specific wire, and no other." That translator is the decoder.

A 2-bit address can name one of 4 things. The decoder turns that address into a one-hot signal: exactly one of sel0..sel3 goes high, the other three stay dark. Toggle a1 and a0 and watch the single lit output march from one wire to the next. It's the smallest primitive that lets a CPU say "pick that one."

the structure: inverters + the AND gates you already know

Look inside and there's nothing new — just gates from earlier pages. On the left, 2 inverters manufacture the opposite polarities ~a1 and ~a0 alongside the originals. On the right, 4 three-input AND gates — the same AND you can build from a NAND plus an inverter — one per output line.

Five vertical buses run between them: a1, ~a1, EN, a0, ~a0. Each AND taps a unique trio — say a1 · a0 · EN, or ~a1 · ~a0 · EN. An AND only fires when all three of its taps are high, and the polarity pattern is wired so that for any address, exactly one AND finds all-highs. That's where the one-hot comes from: four hardwired "is the address exactly this?" tests, run in parallel.

EN is the master switch

Every AND shares one common tap — EN. Click EN to drop it to 0 and watch the whole panel go dark: with that third input low, no AND can ever reach all-highs, so every output is forced to 0 no matter what a1/a0 say. The decoder is muted.

That off switch is exactly why a decoder is safe to embed in a register file. Reading happens every cycle, but you only want a write to land on the addressed register during the cycles you actually mean it. Hold EN=0 and no sel line can raise a write-enable; raise EN=1 for one cycle and precisely one register becomes writable. You'll see this exact gate guarding the writes two pages up.

walk the truth table

Cycle the address through 00 → 01 → 10 → 11 and watch the lit line step sel0 → sel1 → sel2 → sel3, one moving highlight, never two at once. Each address lights the one AND whose polarity trio it matches — the decoder is just four address-equality tests stamped into wiring.

And it scales without changing shape. RISC-V names one of 32 registers with a 5-bit address, so its decoder is 5 inverters and 32 five-input ANDs. Same circuit, more rows.

why this matters

The decoder is the bridge between two worlds that never quite touch. Software thinks in compact numbers because numbers fit in registers and instructions; hardware acts through dedicated wires because a transistor only knows one signal line. Every time those worlds meet, a decoder sits in the seam.

You'll meet it again almost immediately. The very next page, the MUX, hides one inside itself to pick which input to forward. The register file uses one to pick which cell to write. Memory uses one to pick which row to open, and the instruction decoder uses one to pick which operation to run. The one-hot you're toggling here is how a computer turns a name into an action.
