where we are: memory you can write

This is mem with a second door. The read path is exactly the page you already built — a 2-bit address picks one of four cells and the read MUX forwards it as rdata, no clock. What's new is the write path along the bottom: a write port that, on a clock edge, stores a value into one cell.

That's the difference between instruction memory (read-only, on the fetch path) and data memory (read and write, on the load/store path). A load reads a cell; a store writes one.

read a cell (load)

Walk the address through 00, 01, 10, 11.

The addressed cell lights and its bit rides through the read MUX out as rdata — instantly, no clock. This is the load path: give the address, get the stored value back. The other three cells sit untouched.

write a cell (store)

Set wdata 1, point the address at 01, turn we 1, then press ↻ clock.

On the clock edge the write port's decoder routes wdata into the one addressed cell and it flips to 1 — and stays, even after you drop we. Storing needs all three to line up: an address, we = 1, and a clock edge. With we = 0 the clock does nothing — that's how a load leaves memory unchanged.

why this matters

One block, two ports. The read port makes data available the instant you address it; the write port commits a new value on a clock edge. Address it with an ALU result and this is the data memory a load reads and a store writes.

Widen the cell from 1 bit to a 32-bit word and add address bits and this is real RAM. Next we wire it onto the datapath, beside the register file, so loads and stores have somewhere to go.
