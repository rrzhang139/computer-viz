where we are: the counter's loop, with a switch in it

On the program-counter page, the loop was sealed: register → +1 adder → register, forever. Here the loop runs through one new block before it reaches the register — a PC-source MUX. A MUX is a switch: several inputs, one output, and a select wire that decides which input passes through to the output.

The MUX's input 0 carries PC + 1, the sequential future. Input 1 carries the branch target, delivered from the instruction decoder. The select wire is PCSrc — the branch decision, computed beside the ALU as Branch AND Zero.

both futures are always computed

Nothing here waits for permission. The adder is always outputting PC + 1; the target wires always carry whatever the instruction's target field holds. Both candidate futures sit at the MUX's inputs at all times.

Click t₁ to change the target and watch the target readout follow. Notice the MUX picks readout does not change — the target wire being connected causes nothing by itself. The select decides; the inputs just wait.

flip the select

Click PCSrc. The MUX picks readout switches from the +1 value to the target value. Hover the select block to watch the real MUX inside: a different AND row now carries the signal to the OR.

Nothing has been stored yet — the register still holds the old PC. The MUX only chose what is staged at the register's door.

pulse the clock — the chosen future becomes the present

Press pulse CLK 0→1→0. On the rising edge, the register captures whatever the MUX passed through. If PCSrc was 1, the PC is now the target — it leapt. If it was 0, the PC advanced by one — it walked.

Either way the new PC leaves on the address wires to fetch the next instruction, and loops back into the adder so the next +1 is ready. Same loop as the counter — with a decision in it.

why this matters

This is the fetch-front of every real CPU: a 32-bit PC register, a +4 adder, and a PC-source MUX steered by PCSrc = Branch AND Zero. Jumps, function calls, and exceptions each add another input to this same MUX — more futures, same switch.

And the timing story carries over exactly: the adder and MUX are combinational (always computing), the register is sequential (deciding when). The branch CPU embeds this page inside its PC block — what you see here is literally what is running there.
