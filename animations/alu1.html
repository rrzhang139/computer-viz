where we are: the ALU, one bit wide

The ALU is the CPU's calculator: hand it two operands and a 2-bit op and it adds, ANDs, ORs, or XORs them. We build it one bit wide — the true design unit — because a wider ALU is just this slice replicated, one per bit.

A slice takes one bit of each operand (A, B), a carry-in Cin, and the 2-bit op, and produces one result bit Y plus a carry-out Cout. Inputs on the LEFT, results on the RIGHT. This is the same "build the 1-bit cell, then replicate" move you saw with the full adder → 4-bit adder.

four tiny machines, computed at once

All four blocks run together on the same A and B: the full adder computes A + B + Cin, and the single AND, OR, XOR gates each compute their one-bit result. Four answers, ready before op has chosen anything.

Click the full adder to drill into /fulladder.html — it's literally the full adder you built earlier, now doing the ADD inside the ALU.

op picks one answer

The 2-bit op drives a 4-to-1 MUX: 00 → A+B, 01 → AND, 10 → OR, 11 → XOR. Whichever it points at flows to Y; the other three are dropped. Click the MUX to drill into /mux.html — the exact primitive doing the selecting.

Set A = 1, B = 1. With op = 00 the adder wins: Y = 0 and Cout = 1 (1+1 = 10). Flip op0 to 01 — now Y = 1 (1 AND 1), and the carry is ignored.

the carry is the one wire between slices

Only the ADD path uses Cin and Cout. The bitwise ops compute a carry too, but the MUX never selects it, so it's harmless. Toggle Cin with op = 00: Y flips, because the slice is now adding three 1-bit numbers.

This Cout is what feeds the next slice's Cin when you stack these into a wider ALU. Control (op) is broadcast to every slice; data (A, B) is per-slice; the carry is the only wire that runs between slices.

why this matters

This one slice is the true design unit of an ALU. Stack 4 of them (broadcasting op, chaining Cout → Cin) and you have a 4-bit ALU; stack 32 and you have the RV32I ALU that runs add x3, x1, x2.

Widening a datapath is never "redesign the logic" — it's "replicate the bit-slice and broadcast the control." The same idea widened the adder and the register file. The slice is the atom; the width is just how many you place side by side.
