where we are: the whole loop

Every page so far built one part. This wires two of them — the register file and the ALU — into the loop a CPU actually runs.

One instruction, add x3, x1, x2, is four moves: read x1, read x2, add them, write x3. Follow it left to right across the diagram.

step 1 — read two registers

Set read x1 and read x2 to two different registers. The file has two read ports, so both values come out at once — that's why add needs only one pass.

Watch rdataA and rdataB light as they cross into the ALU. No clock — reading is instant.

step 2 — pick the operation

The op bits tell the ALU what to do: 00 add, 01 AND, 10 OR, 11 XOR.

Leave it on 00 for add. The ALU combines its two operands and the answer appears on Y → result.

step 3 — write it back

See Y loop around the bottom into the file's write port. Set write x3 to a target and turn we = 1.

Press ↻ run cycle: on the clock edge Y latches into that register. The result is now stored — a later read can use it. That feedback is what makes a machine make progress.

why this matters

This is the heart of a CPU. Read → operate → write-back is the shape of nearly every instruction.

Widen the wires to 32 bits and drive op/raddr/waddr from a decoded instruction instead of buttons, and this same loop runs RV32I. Everything below it was built so this loop could exist.
