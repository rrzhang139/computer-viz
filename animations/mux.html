where we are: pick one of many

The multiplexer takes N inputs and a binary select, and routes the selected input to a single output. Toggle s1 and s0 to change which input lights up the output wire.

This is the dual of the decoder. Decoder takes an address and produces one-hot wires. MUX takes one-hot wires (from a decoder!) and uses them to gate exactly one input through.

the structure: decoder + ANDs + OR

The block in the upper-center is a literal 2-to-4 decoder (click it to drill into /decoder.html). It takes (s1, s0) and produces 4 one-hot lines.

Each line feeds into one of the 4 AND gates below. Each AND's OTHER input is one of the 4 data inputs. So when sel0=1 (i.e. select=00), only the AND for in0 can pass its value through. The other 3 ANDs see a 0 from their decoder line and output 0.

All 4 AND outputs feed a 4-input OR, which collapses them into a single wire. Since only one AND is active, the OR's output equals whichever input is selected.

walk all 4 selects

Set all 4 inputs to 1 (so any selection produces a 1 output). Now cycle s1, s0 through 00, 01, 10, 11. The lit AND moves bit by bit; the output stays 1.

Now flip in2 to 0 and select 10 (s1=1, s0=0). The output drops to 0 — because the selected input is 0. Pick any other select and the output rises again. The MUX is showing you the value at the addressed input.

the duality with the decoder

You just used the decoder TWICE in different roles: the page /decoder.html shows it standalone — address-in, one-hot-out. Here it sits embedded inside the MUX, producing the same one-hot — but the MUX then uses that one-hot to GATE inputs.

Decoder: "I have an address, light up one of N wires."
MUX: "I have one of N values to pick — use the decoder's one-hot as the gate." Same primitive, opposite role.

why this matters

A CPU is mostly MUXes. The register file's read port is a 32-to-1 MUX picking one of 32 register outputs. The ALU result selector is a MUX picking add/sub/and/or based on opcode. The next-PC selector is a MUX picking sequential vs branch vs jump. Even instruction memory's "which word to fetch" is a giant MUX wrapped in addressing logic.

You've built a 4-input version. Scale the inputs to 32 and the select to 5 bits, and it's the read port of an RV32I register file.
