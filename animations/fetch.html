where we are: where instructions come from

Every page so far built a part that does work once. This page builds the loop that makes a machine run a program: fetch. A program counter (PC) holds the address of the current instruction; the instruction memory hands back the word living there; and on every clock the PC steps +1 so the next cycle fetches the next instruction.

The PC on the left is just a counter — a register with a +1 self-loop (drill into the counter page to see its guts). Its value drives the memory's address. The memory is the mem block you just built (hover it for the real table; click to drill in). Press ↻ clock and watch the PC advance and a new instruction come out.

address out, instruction back

At reset the PC is 00, so it addresses row 0 of memory, and the word there — 00 00 01 10 — rides out as the instruction. The decoded readout reads it as ADD r2, r0, r1. The PC didn't compute anything; it just pointed, and memory answered. That's a fetch.

Press ↻ clock — fetch next.

Two things happen on the edge: the PC ticks 00 → 01 (its +1 loop), and the address it now presents pulls a different word out of memory — the next instruction, OR r3, r2, r0. Read is instant and combinational; the only thing that waited for the clock was the PC advancing.

run the program

Keep clocking. Watch the PC walk 00 → 01 → 10 → 11 and a different instruction appear each step.

That's a program executing — one fetch per clock, the PC marching through memory in order. After 11 it wraps back to 00 (our toy memory holds four instructions; a real PC just keeps counting up through a much larger memory). Hover the instruction memory to watch the addressed row light up inside as the PC moves.

why this matters

This is the engine's idle: the smallest loop that makes a CPU go. Fetch hands a fresh instruction to the rest of the machine every cycle, forever. Everything you built below — the register file, the ALU, the datapath — sits downstream of this, executing whatever word fetch delivers.

One wiring step remains to close the circle: take this instruction, slice it into fields, and drive the datapath's control inputs with them (read these registers, do this op, write that register). That's the decode, and wiring fetch → decode → datapath is the single-cycle CPU — the capstone.
