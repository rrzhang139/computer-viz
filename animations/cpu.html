the whole machine — and you can open every box

This is a working processor's datapath, assembled from pages you've already built. Every block is a live door — hover one (PC, instruction memory, decode, register file, ALU, write-back) to swap it for an exact copy of the page that builds it; click to drill to the transistors.

There are a lot of wires, so each is colored by what it carries — and every step tells you the one or two colors to watch and to ignore the rest: address, instruction, op, operands A + B, the result, the write-back loop, the clock.

Press ↺ reset. The readout above should now read PC 00 · r0=1 r1=1 r2=0 r3=0. We'll press ↻ run instruction once per step, parking the camera on one stage each press.

fetch: PC → instruction memory

Watch two colors only: address (blue) and instr (violet). The PC holds the current address; the blue address wires leave its right side and wrap to the instruction memory, which hands the word back on the violet instr bus rising in the gap to its right.

Press ↻ run instruction once. This is instruction 0, ADD r2,r0,r1. Watch the blue address wires and the violet instr bus light, and the readout's PC tick 00 → 01.

(Hover PC or instruction memory to open the real register cells and the memory's read MUX.)

decode: split the instruction, read two registers

The 8-bit instr bus (/8) enters the decode block, which slices it into four 2-bit field buses (/2). rs1 → the register file's read-A address and rs2 → read-B; rd → the write address; and the amber op bus runs to the ALU. No logic — pure field extraction.

Press ↻ run instruction once. The readout now shows AND r3,r0,r1 — op=01, rs1=00→r0, rs2=01→r1 — so the A/B readout reads A=1 B=1.

(Hover decode to watch the 8-bit word split into its opcode/rs1/rs2/rd fields, each leaving as a /2 bus.)

execute: the ALU computes

Follow the two operands across the central gap into the ALU: green A and teal B. The amber op line from the decoder tells the ALU which operation to forward as the orange result.

Press ↻ run instruction once. The readout now shows OR r2,r0,r1. With green A=1 and teal B=1, watch the result readout become 1 (1 OR 1 = 1).

(Hover the ALU — the adder and the AND/OR/XOR gates all compute at once; the op-MUX forwards the chosen one.)

write-back: the result returns to storage

Follow the pink write-back loop along the bottom. The orange result enters the write-back MUX, then the pink write-back wire carries it all the way back to the register file's write port. The cyan clock edge is what makes that write actually commit to register rd.

Press ↻ run instruction once. The readout shows XOR r3,r0,r1; follow the pink write-back loop and watch the regs readout flip r3 from 1 → 0 (1 XOR 1 = 0). Keep pressing to loop the four-instruction program.

(Hover write-back to open the MUX that picks ALU-result vs. memory-data.)

why this matters

You're looking at a complete CPU datapath — fetch → decode → execute → write-back — and every block opens into the real circuit underneath, down to transistors. Widen the wires to 32 bits and swap the toy ops for the full RV32I set and this shape runs real programs.

The one thing this single-cycle view leaves out is overlap: a pipeline inserts registers between these stages so several instructions run at once. That's the timing layer on top of exactly this datapath.
